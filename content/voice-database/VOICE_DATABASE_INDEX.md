# Voice Database Index

## Purpose
This database is Claude Code's reference for writing scripts in Jay's voice. It exists because generic script-writing produces generic output — Claude Code needs concrete examples of how Jay actually speaks, structures stories, and builds tension.

## The Two-Tier System

### Tier 1 — Voice Reference (`voice-reference/`)
Scripts that Jay and his mentor have reviewed and signed off on. These represent how Jay sounds when the writing is dialed in.

**Use for:** Voice matching, phrasing, verbal tics, tone, rhythm, hook style.
**Do NOT use for:** Structural copying. These scripts have not been performance-validated yet.

**Entry criteria:**
- Jay wrote it (not Rook, not Claude Code from scratch)
- Reviewed with mentor
- Mentor-approved as representative of Jay's voice

### Tier 2 — Proven Winners (`proven-winners/`)
Scripts that have been filmed, posted, and hit performance thresholds.

**Use for:** BOTH voice matching AND structural blueprinting. These are the gold standard.

**Entry criteria (must hit ONE):**
- Skip rate under 40%
- Views at 3x or more of the week's median (breakout)
- Shares or saves significantly above account baseline
- Follow conversion above account baseline

When a voice-reference script gets filmed and hits these thresholds, it gets promoted to proven-winners with performance data attached.

## How Claude Code Uses This Database

When writing a new script:
1. Load the voice-markers.md and structural-patterns.md files for aggregated patterns
2. Load the specific series script(s) from voice-reference/ matching the series being written
3. Match voice precisely — the verbal tics, phrasings, and rhythm should be indistinguishable from the reference scripts
4. For structural decisions, prioritize proven-winners/ over voice-reference/ once available
5. Never copy a script wholesale — use voice-reference for HOW Jay speaks, use creator blueprints from the broader pipeline for WHAT structures work

## Maintenance Rules

- Every new mentor-reviewed script goes in voice-reference/ with extraction notes
- Every filmed script gets evaluated against proven-winners criteria after 48 hours
- Promoted scripts move from voice-reference/ to proven-winners/ with performance data attached
- Scripts that flop stay in voice-reference/ — they're still valid voice references even if the structure didn't land
- Update this index whenever scripts are added, promoted, or the tier criteria change
